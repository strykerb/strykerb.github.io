Program Overview:

This program allows the user to view dynamically generated cylinders in VR, using the WebXR library. The use of a smart phone is required to view the scene. The scene has been tested to work on an iPhone 8 in Safari.

User Guide

  • Select the Enter WebXR button (only selectable on WebXR-Compatible devices.)
  • Rotate the device so that it's oriented horizontally
  • Ideally, view the scene through Google Cardboard or a similar mobile phone HMD solution.


Implementation Notes

The program was based off of A WebXR Example provided by the Mozilla Web Docs. The scene has ambient and diffuse lighting from a directional light source. Smooth Shading was achieved using Gouraud Shading.



Op-Ed

My program: The Fall of The Pump Shotgun, did not go as planned.

Looking back on the Project, I definitely overcommitted myself in terms of project scope. I wanted to use AR and plane detection to place a 3D model into the real world, and as such I chose to use the WebXR library. This, as it turns out, was a huge mistake. I did not find ample documentation to achieve my goals, and I ultimately did not achieve my minimum project specs.

I have worked with WebXR in the past, which contributed to my confidence going into this project, but those past projects also used other high level graphics libraries (Unity) that greatly streamlined and automated much of the work I was doing in JavaScript. While my scene is technically VR, I was unable to test the effect of head movement on the scene, since my development website does not have https, which is required in order for Safari to grant the website sensor data. In theory, moving your head in the scene should allow you to look away from the floating purple crystal.

My main struggle was being unable to get the cylinders to render properly. They ended up looking like spiky crystal structures, so I just went with it. This is likely an issue with the two potentially conflicting Matrix Math libraries and/or something to do with the XR projection matrix, as the cylinder class itself remained untouched from previous assignments. I was attempting to merge two code bases and get them to play nice with each other, and I realized too late that I was in way over my head.

If I were to attempt this project again, I would rewrite the Cylinder class to use the new Matrix Math library, removing the library conflict that caused me so much frustration. I would also do more research on how the WebXR library computes view matrices.
